# Security Policy

## Supported Versions

Only the latest released minor version of SFSymbolsKit is supported with security fixes. Update to the most recent tag before reporting a vulnerability.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Use GitHub's [private vulnerability reporting](https://github.com/WikipediaBrown/SFSymbolsKit/security/advisories/new) to disclose privately. If that is not available to you, email `spook@spookylabs.ai` with:

- A description of the issue
- Steps to reproduce
- The version (or commit SHA) you tested against
- Any suggested mitigation

You should receive an acknowledgement within a few business days. Once the issue is resolved, you'll be credited in the advisory unless you request otherwise.

## Out of scope

Dependabot alerts on this repository's existing dependencies are tracked separately and do not need to be reported. The vulnerabilities surfaced by Dependabot live at <https://github.com/WikipediaBrown/SFSymbolsKit/security/dependabot>.
